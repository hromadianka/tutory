Веб-розробка

Розробка динамічних сайтів (веб-застосунків).

Основи

У веб-застосунку, а відповідно і у веб-розробки є дві частини.

Клієнтська сторона сайту або веб-застосунку (Front end) - це операції, які виконуються на комп’ютері користувача.

Серверна сторона (Back end) - операції, які виконуються на сервері. В основному це операції, пов’язані з базами даних. Наприклад, коли ти реєструєшся на сайті, твій комп’ютер звертається до сервера, щоб вписати в загальну базу даних твій логін і пароль. За серверну сторону можуть відповідати такі технології, як PHP, деякі фреймворки JavaScript (хоча чистий JavaScript відноситься до клієнтської сторони), фреймворки Python - Django і Flask тощо.

No code

No code інструменти дозволяють створювати складні і динамічні веб-застосунки без програмування. Вони досить прості і інтуїтивно зрозумілі.

Корисно ознайомитись з цими інструментами, щоб зрозуміти логіку створення веб-застосунків. Цю логіку можна описати приблизно так: на сайті є статичні елементи (які не змінюються і стосуються фронт-енд частини), а є динамічні. Наприклад, коли користувач Тьюторі відкриває свою сторінку профіля, він бачить там своє ім’я. Щоб це працювало, сайт звертається до серверу і бере ім’я там. У фронт-енд коді цей елемент записаний як перемінна, а що замість неї підставити - описано в бек-енд коді.

Отже, ім’я користувача на сторінці профіля - це динамічний елемент.

Не будь-який сайт містить у собі динамічні елементи. Сайт, який їх містить, і називається “веб-застосунок”.

  • Подивись на сторінку будь-якого веб-застосунку. Які елементи на ньому статичні, а які - динамічні?

  • Bubble

    Навчання на їхньому сайті в розділі Academy.

    Дозволяє створювати веб-застосунки без коду.

  • Glide

    Дозволяє створювати веб- і мобільні застосунки на основі таблиць у Google Sheets, Excel або Airtable без коду.

  • Adalo

    Дозволяє створювати веб- і мобільні застосунки без коду. Навчатися тут

  • Придивись до однієї з цих платформ, розберись, які на ній є інструменти, пограйся з інтерфейсом. Вони схожі на програми для дизайну, як, наприклад, Figma (про неї можна почитати в нашому курсі “Дизайн”), але дозволяють створювати динамічні елементи, прив’язуючи їх до бази даних.

Чому ж нам все ще потрібно програмування, якщо ми можемо використовувати ці інструменти? No code все-таки обмежує. Він не завжди дає можливість реалізувати всі функції, які потрібні проєкту, і приходиться шукати обхідні шляхи або відмовлятися від деяких задумок. Крім того, це платні інструменти, - некомерційному сектору або малому бізнесу вони можуть бути недоступні.

Front-end розробка: HTML і CSS

HTML - це не мова програмування, а мова розмітки. Що це означає? На відміну від мов програмування, HTML не дозволяє писати алгоритми, а просто дає можливість “закодити” такі елементи, як заголовок, картинка, текстовий параграф, форма з полями вводу або кнопка (та інші).

//шаблон хтмл документа, що таке тег і як він виглядає, тег який не закривається, властивості тегу. Блоки дів і спан. Хедер і футер

У HTML елементи прописуються за допомогою тегів. Так виглядає відкриваючий тег <h1> (заголовок), а так закриваючий: </h1>. У тега можуть бути властивості, наприклад, властивість href визначає адресу, на яку буде переводити посилання, це виглядає так: <a href=”tutoryproject.herokuapp.com”>Перейди за посиланням!</a>. Існують також такі властивості, як class і id, вони потрібні, щоб було легше стилізувати елементи, про це ми поговоримо пізніше; називати класи і id можна як завгодно, але без пробілів і використовуючи тільки букви, цифри і нижні підкреслення.

Всередину одного тега можна вставляти інші.

От шаблон HTML-документа:

<!doctype html>
<html>
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>Назва сторінки</title>
  <link rel="stylesheet" href="css/styles.css">
</head>
<body>
  <!-- Зміст сторінки -->
  <script src="js/scripts.js"></script>
</body>
</html>

Перша строка і тег HTML визначають тип документу.

У розділі <head> визначаються головні властивості цього документу, наприклад, у тегах <meta> у цьому шаблоні визначається 1) кодування (простіше кажувати, які символи можна використовувати в документі, щоб комп’ютер їх зрозумів; символи кирилиці входить до кодування utf-8). 2) друга стрічка передає в документ інформацію про розмір екрану користувача, щоб потім можна було підлаштовувати стилізацію під великі і маленькі екрани. Тег <link> тут дозволяє прикріпити CSS-файл, про це буде пізніше. Тег <title> визначає назву сторінки; у звичайному житті ти можеш побачити назви сторінок, якщо подивишся на свої відкриті вкладки; наприклад, ця називається просто Tutory.

Всередині тега <body> - основний контент сторінки.

Тег <script> дає можливість прикріпити JavaScript-файл. JavaScript - це вже мова програмування, яка визначає, що сайт буде робити (це все ще Front-end мова, але за допомогою певних інструментів можна писати і Back-end на JavaScript. Втім, на цьому курсі ми будемо писати Back-end іншою мовою програмування).

На початку тега <body> можна вставити <header> (верхня частина сторінки), а наприкінці - <footer> (нижня частина). У <header> можна розміщувати логотип, назву сайту, меню; у <footer> - посилання на правила користування, на відповіді на часті питання тощо.

Основний контент можна варто на блоки за допомогою тегу <div>. Поділ сайту на блоки допомагає в стилізації.

Але як прописувати стилізацію, або простіше кажучи, оформлення елементів? Якщо HTML визначає, які елементи будуть на сайті, то за допомогою CSS описують, як вони будуть виглядати. CSS-стилі можна прописувати у властивості style (<h1 style=”color: red”>Якийсь заголовок</h1>), всередині окремого тегу <style>, або прикріпити окремий файл у розширенні .css (на шаблоні це зроблено в сьомій стрічці, там де <link>).

Як прописують стилі в CSS? Треба обрати тег, клас або айді, обрати властивість і присвоїти їй якесь значення. Наприклад:

// Стилізувати всі заголовки h1 у прив'язаному документі (робить текст червоним і фон білим)
h1 {
	color: red;
	background-color: white;
}

// Стилізувати елементи з класом example (робить текст у цих елементах підкресленим)
.example {
	text-decoration: underline;
}

// Стилізувати елементи з айді example (центрує текст)
#example {
	text-align: center;
}

Для створення правил, які будуть застосовуватися на різних пристроях, використовують тег @media. Наприклад, для того щоб змінити розмір тексту, колір фону, або розміри блоків на маленькому екрані, можна використовувати такий код:

/* Загальні стилі, які будуть застосовуватися на всіх пристроях */
body {
  font-size: 16px;
}

/* Стилі, які будуть застосовуватися на екранах ширше 768px */
@media (min-width: 768px) {
  body {
    font-size: 20px;
  }
}

/* Стилі, які будуть застосовуватися на екранах ширше 1024px */
@media (min-width: 1024px) {
  body {
    font-size: 24px;
  }
}

У цьому прикладі, на екранах ширше 768px розмір шрифту збільшиться до 20px, а на екранах ширше 1024px - до 24px. Таким чином, використання тегу media дозволяє створювати адаптивний дизайн, який буде оптимізований для будь-яких розмірів екранів.

Щоб створити анімацію при наведенні або нажатті на елемент, потрібно використовувати псевдокласи :hover або :active відповідно.

Ось приклад створення анімації при наведенні на кнопку:

button {
  background-color: blue;
  color: white;
  transition: background-color 0.5s ease;
}

button:hover {
  background-color: red;
}

У цьому прикладі ми використовуємо псевдоклас :hover для зміни фонового кольору кнопки на червоний при наведенні курсора миші на неї. Ми також використовуємо властивість transition, щоб зробити зміну кольору плавнішою.

Аналогічно, для створення анімації при нажатті, використовуй псевдоклас :active замість :hover.

Не треба вчити теги і властивості напам’ять. Ми зосередимось на практичних завданнях. Ти самостійно шукатимеш спосіб виконати те чи інше завдання за допомогою наших підказок.

Front-end розробка: React, Chakra UI

Першим кроком є встановлення бібліотеки React. Це можна зробити за допомогою пакетного менеджера NPM або Yarn. Наприклад, якщо використовуєте NPM, запустіть команду npm install react.

Після встановлення React можна створити перший компонент React. Компонент - це основна одиниця розробки з React. Це може бути будь-який елемент, який потрібно відобразити на сторінці, наприклад кнопка, список, форма і т.д.

Для створення компоненту потрібно створити новий файл з розширенням .jsx або .js і написати наступний код:

import React from 'react';

function MyComponent() {
  return (
    <div>
      <h1>Hello, world!</h1>
    </div>
  );
}

export default MyComponent;

У цьому коді ми імпортуємо бібліотеку React, створюємо новий функціональний компонент MyComponent, який повертає HTML-код за допомогою спеціального синтаксису JSX. JSX дозволяє змішувати HTML і JavaScript в одному файлі.

Коли компонент створено, його можна відобразити на сторінці. Для цього потрібно відрендерити його за допомогою методу ReactDOM.render(). Наприклад, у файлі index.js можна написати наступний код:

import React from 'react';
import ReactDOM from 'react-dom';
import MyComponent from './MyComponent';

ReactDOM.render(<MyComponent />, document.getElementById('root'));

У цьому коді ми імпортуємо бібліотеку ReactDOM, ініціалізуємо компонент MyComponent і рендеримо його в елементі з id="root" на сторінці.

Основні принципи React можна пояснити на прикладі створення компонента кнопки, який змінює свій текст при кожному кліку.

  1. Створюємо компонент кнопки за допомогою функції або класу, який повертає JSX-код.
  1. Встановлюємо стандартний текст для кнопки за допомогою стану компонента.
  1. Додаємо обробник події onClick до кнопки, який викликає функцію, що змінює стан компонента.
  1. В функції, що змінює стан, перевизначаємо текст кнопки.
  1. Відображаємо компонент на сторінці за допомогою ReactDOM.render.

Ось приклад коду, який відображає компонент Counter, який використовує стан та метод setState для збільшення числа на 1 при кожному кліку на кнопку:

import React, { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  function handleClick() {
    setCount(count + 1);
  }

  return (
    <div>
      <p>You clicked {count} times</p>
      <button onClick={handleClick}>Click me</button>
    </div>
  );
}

У цьому прикладі ми використовуємо useState, щоб створити змінну count та метод setCount, який дозволяє оновлювати її значення. Далі ми створюємо функцію handleClick, яка збільшує значення count на 1 та передається як обробник події для кнопки. Нарешті, ми відображаємо значення count та кнопку, яку можна натискати для збільшення лічильника.

Chakra UI - це бібліотека компонентів для React, яка дозволяє швидко створювати дизайн інтерфейсу для веб-сайтів.

  1. Встановіть Chakra UI у свій проект за допомогою команди npm install @chakra-ui/react.
  1. Імпортуйте компоненти з Chakra UI до свого файлу компонентів.
  1. Використовуйте компоненти з бібліотеки у своїй сторінці, наприклад:
import { Button, Input } from "@chakra-ui/react";

function MyPage() {
  return (
    <div>
      <Input placeholder="Введіть текст" />
      <Button colorScheme="blue">Надіслати</Button>
    </div>
  );
}
  1. Налаштуйте стиль компонентів з допомогою пропсів, наприклад:
<Button colorScheme="green" size="lg" >
  Натисніть мене
</Button>
  1. Налаштуйте тему оформлення за допомогою ChakraProvider, наприклад:
import { ChakraProvider, extendTheme } from "@chakra-ui/react";

const theme = extendTheme({
  colors: {
    brand: {
      50: "#e3f1f3",
      100: "#c1d9db",
      // додайте більше кольорів
    },
  },
});

function MyApp({ Component, pageProps }) {
  return (
    <ChakraProvider theme={theme}>
      <Component {...pageProps} />
    </ChakraProvider>
  );
}

export default MyApp;

Використовуй документацію Chakra UI, щоб дізнатися про додаткові функції та компоненти.

Back-end розробка: Django

Як розпочати джанго проєкт. Юрл, вид, шаблон і перемінні, модель, форма, аутентифікація. Як прикріпити статичний файл.

Django - це фреймворк Python (якщо дуже спрощено, фреймворк - це бібліотека додаткових команд, які створюють розробники на основі якоїсь мови програмування), націлений на веб-розробку.

Що обов’язково знати про Python, щоб розпочати розробку?

  • Змінні.

Змінні в Python - це просто імена, які ми даємо певним значенням, щоб зберегти їх та використовувати пізніше в програмі. Наприклад, якщо ми хочемо зберегти число 10, ми можемо створити змінну з іменем x та присвоїти їй значення 10:

x = 10

Тепер, коли ми згадуємо змінну x в нашій програмі, Python розуміє, що ми маємо на увазі число 10. Ми можемо також змінювати значення змінної, просто присвоюючи їй нове значення:

x = 5

Тепер, коли ми використовуємо змінну x, Python буде розуміти, що ми маємо на увазі число 5. Таким чином, змінні дають нам можливість зберігати та маніпулювати значеннями в нашій програмі.

  • Які бувають змінні?

У Python існує кілька типів змінних, які визначають, як дані можуть бути збережені та опрацьовані:

  • Числа: цілі числа (наприклад, 42) та числа з плаваючою крапкою (наприклад, 3.14).
  • Рядки: це послідовність символів (наприклад, "Hello, world!").
  • Списки: це послідовність значень, які можуть бути різного типу. Списки можуть змінюватися, тобто елементи можуть бути додані, видалені або змінені.
  • Кортежі: це послідовність значень, які не можуть бути змінені після створення.
  • Словники: це набір пар "ключ-значення", які можуть бути використані для зберігання та доступу до даних за допомогою ключа.
  • Множини: це набір унікальних значень, які можуть бути використані для операцій над множинами, таких як об'єднання, перетин або різниця.

Кожен тип змінної має свої особливості та операції, які можуть бути виконані з ними. Змінні можна створювати, зберігати та обробляти у програмах, що дозволяє вирішувати багато задач, що вимагають роботи з даними.

Приклад списку:

# Створення списку
fruits = ['apple', 'banana', 'cherry', 'orange']

# Доступ до елементів списку
print(fruits[0])   # Виводиться 'apple'
print(fruits[1])   # Виводиться 'banana'
print(fruits[2])   # Виводиться 'cherry'

# Додавання нових елементів до списку
fruits.append('pear')
print(fruits)      # Виводиться ['apple', 'banana', 'cherry', 'orange', 'pear']

# Видалення елементів зі списку
fruits.remove('banana')
print(fruits)      # Виводиться ['apple', 'cherry', 'orange', 'pear']

Приклад словнику:

# Створення словника
person = {'name': 'John', 'age': 30, 'city': 'New York'}

# Доступ до елементів словника
print(person['name'])   # Виводиться 'John'
print(person['age'])    # Виводиться 30
print(person['city'])   # Виводиться 'New York'

# Додавання нових елементів до словника
person['email'] = 'john@example.com'
print(person)           # Виводиться {'name': 'John', 'age': 30, 'city': 'New York', 'email': 'john@example.com'}

# Видалення елементів зі словника
del person['city']
print(person)           # Виводиться {'name': 'John', 'age': 30, 'email': 'john@example.com'}

Тепер розпочнемо розробку в Django.

Для початку проєкту на Django, потрібно встановити Python та пакетний менеджер pip на свій комп'ютер. Після цього встановлюємо Django за допомогою команди в терміналі:

Після встановлення Django, створюємо віртуальне середовище для проєкту, щоб ізолювати його залежності від інших проєктів на комп'ютері. Для цього створюємо нову папку та переходимо в неї через термінал. Потім виконуємо наступні команди:

// На місці останного venv має бути назва віртуального середовища, можна обрати й іншу

Далі активуємо віртуальне середовище. Для цього треба написати в термінал розташування файлу activate.bat. Наприклад, якщо віртуальне середовище розташовано в диску С, у папці користувача User, треба написати таке:

C://users/User/venv/Scripts/activate.bat

Встановлюємо Django в наше віртуальне середовище.

Тепер ми можемо створити новий проєкт Django за допомогою команди:

Після цього можна перейти в папку проєкту (на Windows це команда cd project_name) та запустити його локально на сервері за допомогою команди:

Це дозволить перейти за адресою http://127.0.0.1:8000/, де буде відображено стандартне повідомлення про успішний запуск проєкту Django.

Тепер починаємо саму розробку.

У файлах твого веб-застосунку можна знайти файл urls.py. Там зберігаються посилання, які ти додав(ла) у веб-застосунок. Наприклад:

urlpatterns = [
    path('admin/', admin.site.urls),
    path('', views.index),
    path ('signup', views.signup),
]

Тобто, коли ти перейдеш просто за посиланням сайт (поки там просто якісь цифри), відкриється вид index. Якщо до цього додати /signup, відкриється вид із назвою signup (можна називати вид не так, як посилання).

Коли ти переходиш за посиланням, викликається вид. Види зберігаються у файлі views.py. Вид - це код на Python, у якому можна виконати будь-який алгоритм і викликати шаблон. Це працює так:

def signup(request):
	return render(request, 'signup.html')

Шаблон - це просто HTML-документ, у якому можна використовувати змінні, інформація з яких підвантажується з баз даних (наприклад, щоб в одному з елементів показувалося ім’я користувача, або щоб у стрічці показувалися тільки пости тих користувачів, на яких ти підписаний). Так виглядає шаблон:

<!DOCTYPE html>
{% load static %}

<html lang="en" xmlns="http://www.w3.org/1999/xhtml">
<head>
    <meta charset="utf-8" content="width=device-width, initial-scale=1" name="viewport" />
    <title>Назва сторінки</title>
</head>
<body>
		<p>{{x}} - це значення змінної x.
</html>

Щоб використовувати змінні у шаблоні, у виді треба визначити цю змінну і записати її в контекст. Ось так:

def signup(request):
	x = 5
	y = x+1
	context = {'x': x, 'y': y}
	return render(request, 'signup.html', context)

Тепер можна використовувати у шаблоні {{x}} і {{y}}, змінна x буде замінюватися на 5, а y - на 6. Звісно, це не дозволяє нам робити так, щоб для різних користувачів сторінка виглядала по-різному. Нам потрібні бази даних.

Як записати щось у базу даних? Треба створити модель. Наприклад, ми створюємо модель користувача Profile. Для цього у файлі models.py пишемо:

from django.contrib.auth import get_user_model
import uuid

User = get_user_model()

class Profile (models.Model):
    user = models.ForeignKey(User, on_delete=models.CASCADE)
    email = models.EmailField()
    description = models.TextField(default='')

    def __str__(self):
        return self.user.username

Що ми зробили? Ми створили в базі даних модель Profile, у якої є поля email і опис (текстове поле). Також ця модель прив’язана до вбудованої в Django моделі User. Це робиться тільки для моделі користувача, але загалом тип поля ForeignKey дозволяє доєднати доєднати до одного екземпляра моделі інші екземпляри іншої моделі. Тоді за допомогою цього поля можна викликати, наприклад, пости, опубліковані тим чи іншим користувачем.

Потім ми:

  • У командній стрічці виконуємо команди: 1) python manage.py makemigrations 2) python manage.py migrate (це треба робити після будь-якої зміни моделей)
  • У видах на початку пишемо: from .models import Profile
  • Додаємо модель на панель адміністратора: у файлі admin.py пишемо: 1) from .models import Profile 2) admin.site.register(Profile)

Щоб увійти на панель адміністратора, треба:

1) Виконати в терміналі команду python manage.py createsuperuser і далі зареєструватися за інструкцією.

2) Додати в базове посилання твого сайту /admin (має вийти щось на кшталт: http://127.0.0.1:8000/admin) і виконати вхід.

Готово, на цій панелі можна вручну створювати і змінювати екземпляри моделей.

Як використовувати модель у сайті?

По-перше, тепер ми можемо прирівнювати якісь перемінні до окремих властивостей моделі, використовуючи команди get і filter.

Наприклад:

@login_required(login_url='/signin')
def signup(request):
	user = User.objects.get(username=request.user.username)
  profile = Profile.objects.get(user=user)
  description = profile.description
	context = {'description': description}
	return render(request, 'signup.html', context)

Що ми зробили? Ми отримуємо дані про користувача, який виконав вхід. Потім отримуємо з бази даних профіль, поле user якого дорівнює нашому користувачу. Після цього отримуємо поле “опис” з цього профілю. Якщо користувач не виконав входу, то його буде автоматично переслано за посиланням /login, там у нас має розташовуватися форма входу. У цьому шаблоні ми тепер можемо використовувати змінну description - тобто опис профілю користувача, який на цей момент виконав вхід.

Так ми можемо вивести всі профілі, опис яких є “Нема що написати”

@login_required(login_url='/signin')
def signup(request):
  profiles = Profile.objects.filter(description='Нема що написати')
  context = {'profiles': profiles)
	return render(request, 'signup.html', context)

Щоб у шаблоні вивести всі ці пости, записуємо:

{% if profiles %}{% for profile in profiles %}<p>{{profile.user.username}}</p><p>{{profile.description}}</p>{% endfor %}{% else %}Таких профілів нема!{% endif %}

Щоб замість цього виводити тільки профілі з описом, який тільки містить речення “Нема що написати”, але не обов’язково повністю дорівнює цьому речення, profiles треба дорівняти до Profile.objects.filter(description__icontains='Нема що написати').

Щоб зробити щось у виді тільки якщо користувач авторизований, записуємо: 

if request.user.is_authenticated:
	//Код, який  буде виконуватись, тільки якщо користувач авторизований

Щоб вивести всі екземляри моделі, є команда Profile.objects.all (таке значення треба присвоїти якійсь змінній і потім її використувавати).

Можна зробити так, щоб користувач міг сам створювати або змінювати екземпляр моделі. Це знадобиться в реєстрації, дозволить користувачу щось публікувати тощо. Для цього ми створюємо два шаблони з HTML-формами. Для кожного поля визначаємо властивість name. У властивостях самого тегу form записуємо request=’POST’. У видах пишемо:

def signup(request):

    if request.method == 'POST':
        username = request.POST['username']
        email = request.POST['email']
        password = request.POST['password']
        password2 = request.POST['password2']

        if password == password2:
            if User.objects.filter(email=email).exists():
                messages.info(request, 'Ця пошта вже зайнята:( Спробуй увійти в акаунт')
                return redirect('/signup')
            elif User.objects.filter(username=username).exists():
                messages.info(request, 'Цей нік вже зайнятий:( Спробуй інший')
                return redirect('/signup')
            else:
                user = User.objects.create_user(username=username, email=email, password=password)
                user.save()

                user_login = auth.authenticate(username=username, password=password)
                auth.login(request, user_login)

                user_model = User.objects.get(username=username)
                new_profile = Profile.objects.create(user=user_model, id_user=user_model.id, email=user.email)
                new_profile.save()
                return redirect('/')
        else:
            messages.info(request, 'Паролі не збігаються! Спробуй ще раз')
            return redirect('/signup')
        
    else:
        return render(request, 'signup.html')

def signin(request):
    
    if request.method == 'POST':
        username = request.POST['username']
        password = request.POST['password']

        user = auth.authenticate(username=username, password=password)

        if user is not None:
            auth.login(request, user)
            return redirect('/')
        else:
            messages.info(request, 'Неправильна пошта або пароль')
            return redirect('/signin')

    else:
        return render(request, 'signin.html')

Таким чином, у виді реєстрації ми приймаємо з форми дані полей з іменами (name) username, email, password, password2, і якщо все добре, створюємо такого користувача, профіль для нього, виконуємо вхід і переспрямовуємо на основну сторінку. Аналогічно у виді входу.

Ти можеш створити базовий шаблон (у якому будуть елементи, які мають бути на кожній сторінці, наприклад, логотип і меню) з назвою base.html, і на початку кожного шаблону писати {{variable1}}, тоді новий HTML-код буде доєднуватися до базового шаблону.

Як зробити вид у деталях? Так, щоб можна було, наприклад, вивести загальний перелік користувачів, а натиснувши на юзернейм, користувач міг би перейти на його сторінку, де буде вся інформація про нього. Для цього в URL прописуємо:

path(profile/<str:pk>', views.profile)

Тепер детальний вид профілю буде розташований за адресою типу такої: site.com/profile/korystuvach1. Але для цього треба ще дещо прописати.

У шаблоні з загальним переліком робимо так, щоб юзернейм був посиланням на сторінку з деталями:

{% for profile in profiles %}<p><a href='/profile/{{profile.user.username}}'>{{profile.user.username}}</a></p>{% endfor %}

Нарешті, прописуємо вид у деталях:

def profile1(request, pk):
    user = User.objects.get(username=pk)
    profile = Profile.objects.get(user=user)
    context = {'profile': profile}
    return render (request, 'profile.html', context)

Готово, ми отримали юзернейм з посилання, і тепер у шаблоні profile.html можна використовувати змінну profile і відповідно властивості профілю.

У загальному переліку можна відобразити, наприклад, тільки перші 20 символів опису профілю за допомогою команди: {{profile.description|truncatechars:80}}

Як додати статичний файл (наприклад, CSS або якісь зображення) у Django?

  1. Створи папку static в директорії твого Django проекту.
  1. У цій папці вашого додатку створи ще одну папку з назвою css, images або будь-якої іншої назви відповідно до типу файлів, які ти додаєш (це необов’язково).
  1. Помісти свої статичні файли у відповідну папку.
  1. У файлі settings.py вашого Django проекту додай STATIC_URL та STATICFILES_DIRS.
STATIC_URL = '/static/'

STATICFILES_DIRS = [
    os.path.join(BASE_DIR, 'static'),
]
  1. У своєму HTML-файлі використовуй тег "{% load static %}" і вказуй шлях до свого статичного файлу використовуючи {{variable}} тег.

Наприклад, якщо в тебе є файл style.css у папці css, ти можеш вказати шлях до нього так:

{% load static %}
<link rel="stylesheet" href="{% static 'css/style.css' %}">
